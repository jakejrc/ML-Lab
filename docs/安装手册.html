第1章 环境要求

1.1 硬件要求

资源最低配置推荐配置说明
CPU双核四核及以上神经网络训练需要较多计算资源
内存4 GB8 GB 及以上大数据集（如手写数字）加载时需要较多内存
硬盘空间2 GB5 GB包含 Python 环境和所有依赖包
GPU不需要可选平台基于 Scikit-learn，主要使用 CPU 计算

1.2 软件要求

软件版本要求说明
操作系统Windows 10/11、Ubuntu 20.04+、macOS 12+支持主流操作系统
Python3.9 ~ 3.11Docker 镜像基于 Python 3.11-slim
pip21.0+Python 包管理工具
Git2.0+（可选）用于克隆项目仓库
Docker20.10+（可选）仅 Docker 部署方式需要

1.3 网络要求

  • 安装阶段：需要互联网连接，用于下载 Python 依赖包
  • 运行阶段（核心功能）：数据工作台、算法实验室、模型评估无需网络
  • 运行阶段（AI 助教）：需要访问通义千问 API 服务，需保持网络畅通

说明

如果不需要 AI 助教功能，平台可以在完全离线的环境中运行（前提是已完成依赖安装）。

第2章 本地安装

2.1 获取源码

方式一：通过 Git 克隆

git clone <仓库地址> ML-Lab
cd ML-Lab

方式二：直接下载

将项目源码压缩包解压到目标目录即可。

项目目录结构如下：

ML-Lab/
├── app.py                  # 主入口文件
├── requirements.txt        # Python 依赖清单
├── Dockerfile              # Docker 部署文件
├── README.md               # 项目说明
├── ml_lab/                 # 核心模块包
│   ├── __init__.py         # 包初始化 (v1.0.0)
│   ├── algorithms.py       # 算法引擎
│   ├── preprocessing.py    # 数据处理
│   ├── evaluation.py       # 模型评估
│   ├── visualization.py    # 可视化
│   └── llm_assistant.py    # AI 助教
├── docs/                   # 文档目录
└── ppt/                    # 演示文稿目录

2.2 创建虚拟环境

强烈建议使用虚拟环境隔离项目依赖，避免与其他项目冲突。

使用 venv（Python 内置）

# Windows
python -m venv venv
venv\Scripts\activate

# Linux / macOS
python3 -m venv venv
source venv/bin/activate

使用 Conda（可选）

conda create -n ml-lab python=3.11
conda activate ml-lab

提示

激活虚拟环境后，命令行提示符前会显示 (venv) 或 (ml-lab) 标识，表示已进入虚拟环境。

2.3 安装依赖

在虚拟环境激活状态下，执行以下命令安装所有依赖：

pip install -r requirements.txt

平台依赖以下核心库：

依赖包最低版本用途
numpy=1.21.0数值计算基础库
pandas=1.3.0数据表处理与分析
scikit-learn=1.0.0机器学习算法库
matplotlib=3.4.0数据可视化绑图
gradio=4.0.0Web 交互界面框架
requests=2.25.0HTTP 请求（AI 助教 API 调用）
torch=1.9.0PyTorch 深度学习框架（可选）

注意

torch 包体积较大（约 2GB），如果仅使用 Scikit-learn 实现的算法，可暂时跳过其安装。如需使用 PyTorch 相关功能，建议使用官方推荐的安装命令，根据本机 CUDA 版本选择对应版本。

国内镜像加速（可选）

如下载速度较慢，可使用国内镜像源：

pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

2.4 配置 API Key

AI 助教功能需要配置通义千问（Qwen）的 API Key。

获取 API Key

  1. 访问阿里云百炼平台（DashScope）
  2. 注册/登录阿里云账号
  3. 进入"API-KEY 管理"页面
  4. 点击"创建新的 API-KEY"，复制生成的 Key

配置方式

在项目根目录设置环境变量或修改源码中的 API Key 配置：

方式一：设置环境变量（推荐）

# Windows (CMD)
set QWEN_API_KEY=your_api_key_here

# Windows (PowerShell)
$env:QWEN_API_KEY="your_api_key_here"

# Linux / macOS
export QWEN_API_KEY=your_api_key_here

方式二：修改源码配置

打开 ml_lab/llm_assistant.py，找到 API Key 配置位置，填入您的 Key。

安全提示

请勿将 API Key 硬编码在代码中并提交到公开仓库。建议使用环境变量或 .env 文件管理敏感信息，并在 .gitignore 中排除 .env 文件。

2.5 启动平台

在项目根目录下执行：

python app.py

启动成功后，控制台会输出 Gradio 访问地址，默认为：

Running on local URL:  http://127.0.0.1:7860

浏览器自动打开该地址即可访问平台。如未自动打开，手动复制地址到浏览器地址栏。

自定义启动参数

参数默认值说明
--port7860指定服务端口
--shareFalse启用 Gradio 公共分享链接（需网络）

示例：指定 8080 端口并启用分享：

python app.py --port 8080 --share

2.6 验证安装

启动平台后，按以下步骤验证各功能模块是否正常：

  1. 界面加载：浏览器正确显示蓝白主题的主界面和四个导航标签
  2. 数据工作台：选择"鸢尾花"数据集，确认数据预览和统计信息正常显示
  3. 算法实验室：使用默认参数训练"决策树"模型，确认训练完成并显示结果
  4. 模型评估：查看评估指标和可视化图表，确认图表正常渲染
  5. AI 助教（可选）：发送一个预设问题，确认 API 调用成功并返回回答

提示

如果某一步验证失败，请参考第 5 章常见安装问题进行排查。

第3章 Docker 部署

Docker 部署适合教学环境批量部署、服务器部署等场景，可避免本地环境配置问题。

3.1 安装 Docker

Windows

  1. 下载 Docker Desktop for Windows
  2. 安装并启动 Docker Desktop
  3. 确认 Docker Engine 运行正常： 
    docker --version
docker run hello-world

Linux (Ubuntu)

sudo apt update
sudo apt install docker.io docker-compose -y
sudo systemctl start docker
sudo systemctl enable docker

3.2 构建镜像

在项目根目录（包含 Dockerfile 的目录）执行：

docker build -t ml-lab:1.0.0 .

Dockerfile 内容说明：

# 基础镜像：Python 3.11 精简版
FROM python:3.11-slim

# 设置工作目录
WORKDIR /app

# 复制依赖文件并安装
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制项目文件
COPY . .

# 暴露 Gradio 默认端口
EXPOSE 7860

# 启动命令
CMD ["python", "app.py"]

说明

构建过程可能需要 5~15 分钟（取决于网络速度），因为需要下载 Python 基础镜像和安装所有依赖包。建议使用国内 Docker 镜像源加速。

3.3 运行容器

基本启动

docker run -d --name ml-lab -p 7860:7860 ml-lab:1.0.0

配置 API Key 启动

docker run -d --name ml-lab \
  -p 7860:7860 \
  -e QWEN_API_KEY=your_api_key_here \
  ml-lab:1.0.0

启动后验证

# 查看容器状态
docker ps

# 查看容器日志
docker logs ml-lab

在浏览器中访问 http://localhost:7860 即可使用平台。

3.4 停止与清理

# 停止容器
docker stop ml-lab

# 启动已停止的容器
docker start ml-lab

# 删除容器
docker rm ml-lab

# 删除镜像
docker rmi ml-lab:1.0.0

第5章 常见安装问题

Q1：pip install 报错 "Python version not satisfied"

原因：当前 Python 版本不在 3.9~3.11 范围内。
解决：安装 Python 3.11，或使用 pyenv/conda 管理多版本 Python。

Q2：安装 scikit-learn 时出现编译错误

原因：缺少 C++ 编译器或构建工具。
解决：

  • Windows：安装 Visual Studio Build Tools
  • Linux：执行 sudo apt install build-essential python3-dev
  • 或直接使用预编译的二进制包：pip install --only-binary :all: scikit-learn

Q3：matplotlib 中文显示为方框

原因：matplotlib 默认不支持中文字体。
解决：平台代码中已配置中文字体支持。如仍有问题，确认系统已安装中文字体（如微软雅黑），或在代码中手动指定字体路径。

Q4：启动后 Gradio 页面空白或报错

可能原因与解决方案：

  • 端口被占用：更换端口 python app.py --port 7861
  • 依赖版本不兼容：尝试升级 gradio pip install --upgrade gradio
  • 浏览器缓存：使用 Ctrl+F5 强制刷新

Q5：Docker 构建非常慢

原因：从 Docker Hub 下载基础镜像速度慢。
解决：配置国内 Docker 镜像加速器（如阿里云、中科大、网易镜像源），修改 Docker 的 daemon.json 配置文件。

Q6：AI 助教功能不可用

排查步骤：

  1. 确认 API Key 已正确配置（检查环境变量或代码配置）
  2. 确认网络可以访问通义千问 API 服务地址
  3. 查看控制台输出的错误信息定位具体原因
  4. 如 API Key 已过期，需重新获取

Q7：内存不足 (OOM) 导致训练失败

原因：使用大数据集（如手写数字 1797 样本 x 64 特征）且同时开启多项可视化时可能占用较多内存。
解决：关闭不必要的程序释放内存，或选择较小的数据集（如鸢尾花）进行实验。
