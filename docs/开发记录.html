第1章 项目概述

1.1 项目背景

ML-Lab（Machine Learning Laboratory）是为机器学习教学和实验而开发的可视化实验平台。项目旨在为职业院校和高校的机器学习课程提供一个零代码门槛、全流程可视化的实验环境，让学生无需编写代码即可体验数据预处理、算法训练、模型评估的完整流程。

同时，项目集成了大语言模型（通义千问）作为 AI 助教，为学生提供实时的概念解释和学习辅导，形成"做中学"与"问中学"相结合的教学模式。

1.2 设计目标

目标描述实现方式
零代码操作用户无需编写任何代码即可完成机器学习全流程实验Gradio 交互式界面
全流程覆盖从数据加载到模型评估，完整覆盖机器学习工作流四大功能页面：数据工作台、算法实验室、模型评估、AI助教
多算法支持涵盖分类和回归两大类任务的经典算法统一注册表封装 6 大算法
可视化驱动通过丰富的图表帮助用户理解模型行为10+ 种可视化图表
AI 辅助教学集成大语言模型提供智能问答通义千问 API + 多轮对话管理
易于部署支持本地安装和 Docker 容器化部署Dockerfile + requirements.txt

第2章 系统架构

2.1 整体架构

ML-Lab 采用分层架构设计，从上到下分为：

  1. 表现层 (Presentation)：app.py — Gradio Web 界面，负责用户交互和结果展示
  2. 业务逻辑层 (Business Logic)：ml_lab/ 包 — 各功能模块的具体实现
    • 算法引擎 (algorithms.py)
    • 数据处理 (preprocessing.py)
    • 模型评估 (evaluation.py)
    • 可视化 (visualization.py)
    • AI 助教 (llm_assistant.py)
  3. 基础层 (Foundation)：第三方库 — Scikit-learn、NumPy、Pandas、Matplotlib、Requests

2.2 模块划分

模块职责对外接口依赖
app.py界面编排、事件处理、状态管理Gradio Blocks所有 ml_lab 子模块
algorithms.py算法实例化、训练、预测ALGORITHM_REGISTRY, create_algorithm()scikit-learn
preprocessing.py数据加载、清洗、缩放、划分load_dataset(), inject_missing(), handle_missing(), scale_features(), split_data()sklearn.datasets, pandas, numpy
evaluation.py评估指标计算、报告生成evaluate_classification(), evaluate_regression()sklearn.metrics
visualization.py各类图表生成10+ 个绑图函数matplotlib, sklearn
llm_assistant.pyLLM API 调用、对话管理call_qwen_api(), ConversationManagerrequests

2.3 数据流

平台的核心数据流如下：

  1. 用户在数据工作台选择数据集 → preprocessing.py 加载数据 → 展示预览和统计信息
  2. 用户配置预处理参数 → preprocessing.py 执行缺失值处理/特征缩放 → visualization.py 生成对比图
  3. 用户划分数据集 → preprocessing.py 返回训练集和测试集
  4. 用户在算法实验室选择算法和参数 → algorithms.py 通过工厂函数创建算法实例 → 调用 train() 训练模型
  5. 训练完成后 → evaluation.py 计算评估指标 → visualization.py 生成结果图表
  6. 用户在AI 助教提问 → llm_assistant.py 构建请求 → 调用通义千问 API → 返回回答

第3章 模块设计详解

3.1 主界面模块 (app.py)

app.py 是平台的入口文件（约 27KB），负责构建完整的 Gradio Web 界面。

核心设计

  • Gradio Blocks：使用 Gradio Blocks API 构建自定义布局，而非简单的 Interface
  • 四页面架构：通过 CSS 显隐控制实现单页应用的标签切换效果
  • 状态管理：使用 Gradio 的 gr.State() 组件在页面间共享数据（如加载的数据集、训练好的模型）

页面组成

页面Gradio 组件核心功能
数据工作台Dropdown, Slider, RadioGroup, Button, DataFrame, Plot数据集选择、预处理配置、数据探索
算法实验室Dropdown, Slider, Textbox, Button, Text, Plot算法选择、参数调节、模型训练
模型评估Button, Text, Plot评估指标展示、可视化图表
AI 助教Chatbot, Textbox, Button, Dropdown对话交互、预设问题

3.2 算法引擎模块 (algorithms.py)

algorithms.py 实现了 6 大机器学习算法的面向对象封装。

设计模式

采用策略模式 + 工厂模式：

  • 每个算法封装为一个独立类，实现统一的接口方法：train()、predict()、get_params()
  • 通过 ALGORITHM_REGISTRY 字典注册所有可用算法
  • 通过 create_algorithm() 工厂函数根据名称创建算法实例

算法类一览

类名任务类型Scikit-learn 实现关键特性
LinearRegressionModel回归LinearRegression最小二乘法，支持正则化
LogisticRegressionModel分类LogisticRegression支持多分类，可配 C/solver
KNNModel分类/回归KNeighborsClassifier/Regressor可调 K 值、权重、距离度量
DecisionTreeModel分类/回归DecisionTreeClassifier/Regressor可调深度、分裂准则
SVMModel分类/回归SVC/SVR支持多种核函数
NeuralNetworkModel分类/回归MLPClassifier/MLPRegressor可配隐藏层、激活函数

统一接口设计

class BaseAlgorithmModel:
    def train(self, X_train, y_train): ...  # 训练模型
    def predict(self, X_test): ...          # 预测
    def get_params(self): ...               # 获取当前参数

3.3 数据处理模块 (preprocessing.py)

preprocessing.py 负责数据集管理和预处理流水线。

内置数据集管理

通过 datasets_map 字典管理 6 个内置数据集的加载器配置：

  • 分类数据集：鸢尾花、葡萄酒、乳腺癌、手写数字
  • 回归数据集：合成分类 (Moons)、合成回归

核心函数

函数功能关键参数
load_dataset()加载指定数据集dataset_name: 数据集名称
inject_missing()在数据中注入缺失值ratio: 缺失值比例
handle_missing()处理缺失值strategy: drop/mean/median/mode
scale_features()特征缩放method: standard/minmax/maxabs/robust
split_data()数据集划分test_size, random_state
get_summary()统计摘要返回 DataFrame 统计信息

3.4 模型评估模块 (evaluation.py)

evaluation.py 提供分类和回归两大类任务的评估能力。

分类评估

evaluate_classification(y_true, y_pred, y_prob=None)

计算并返回以下指标：

  • 准确率 (Accuracy)
  • 精确率 (Precision) — 宏平均
  • 召回率 (Recall) — 宏平均
  • F1 分数 — 宏平均
  • 分类报告 (Classification Report) — 每类别的详细指标

回归评估

evaluate_regression(y_true, y_pred)

计算并返回以下指标：

  • 均方误差 (MSE)
  • 均方根误差 (RMSE)
  • 平均绝对误差 (MAE)
  • 决定系数 (R²)

3.5 可视化模块 (visualization.py)

visualization.py 提供了 10+ 种可视化函数，覆盖数据探索、训练过程和结果分析全流程。

函数图表类型应用场景
plot_data_distribution()数据分布图数据探索阶段，了解特征分布
plot_preprocessing_comparison()预处理前后对比图对比缺失值处理/特征缩放效果
plot_decision_boundary()决策边界图展示分类器在二维特征空间的决策区域
plot_training_history()训练历史曲线展示损失/准确率随迭代次数的变化
plot_confusion_matrix()混淆矩阵热力图分类结果详细分析
plot_roc_curve()ROC 曲线评估分类器在不同阈值下的表现
plot_feature_importance()特征重要性图分析各特征对模型预测的贡献度
plot_regression_results()回归拟合图真实值 vs 预测值散点对比
plot_gradient_descent_2d()梯度下降 2D 可视化展示梯度下降优化过程

设计原则

  • 所有绑图函数返回 matplotlib.Figure 对象，与 Gradio 的 gr.Plot 组件兼容
  • 统一设置中文字体支持，避免图表中文乱码
  • 配色风格一致，使用蓝白主题色系
  • 自动适配分类/回归任务类型，选择合适的图表

3.6 AI 助教模块 (llm_assistant.py)

llm_assistant.py 实现了基于通义千问大语言模型的智能问答功能。

核心组件

1. API 调用函数

call_qwen_api(prompt, conversation_history=None, system_prompt=None)
  • 向通义千问 API 发送请求
  • 支持传入对话历史实现上下文连贯
  • 支持自定义系统提示词
  • 返回模型生成的文本回答

2. ConversationManager 对话管理器

  • 维护多轮对话历史
  • 自动拼接系统提示词 + 历史消息 + 当前问题
  • 支持清空对话历史
  • 限制历史长度，避免上下文过长

3. 系统提示词 (SYSTEM_PROMPT)

定义 AI 助教的角色和行为准则：

  • 角色定位：机器学习教学助手
  • 内置知识库：6 个 ML 核心概念领域
  • 回答风格：清晰易懂，适合初学者，使用中文
  • 回答格式：结构化、分段落、善用示例

4. 预设问题列表

预置 8 个高频问题，覆盖过拟合、算法选型、交叉验证、特征工程、偏差-方差权衡、不平衡数据、正则化、梯度下降等核心知识点。

第4章 算法注册表设计

4.1 注册表结构

算法注册表 ALGORITHM_REGISTRY 是一个字典，键为算法名称，值为算法配置信息：

ALGORITHM_REGISTRY = {
    "linear_regression": {
        "name": "线性回归",
        "class": LinearRegressionModel,
        "type": "regression",
        "description": "使用最小二乘法拟合线性关系",
        "default_params": { ... }
    },
    "logistic_regression": {
        "name": "逻辑回归",
        "class": LogisticRegressionModel,
        "type": "classification",
        ...
    },
    ...
}

注册表字段说明

字段类型说明
namestr算法的中文显示名称
classtype算法实现类的引用
typestr算法类型："classification" 或 "regression"
descriptionstr算法简要描述
default_paramsdict默认超参数配置

4.2 工厂函数

create_algorithm() 工厂函数根据算法名称创建实例：

def create_algorithm(algorithm_name, **kwargs):
    if algorithm_name not in ALGORITHM_REGISTRY:
        raise ValueError(f"未知算法: {algorithm_name}")
    config = ALGORITHM_REGISTRY[algorithm_name]
    params = {**config["default_params"], **kwargs}
    return config["class"](**params)

工厂函数的设计优势：

  • 调用方无需知道具体类名，只需传入算法标识符
  • 自动合并默认参数与用户自定义参数
  • 集中管理算法的创建逻辑，新增算法只需更新注册表

4.3 扩展新算法

如需新增算法，按以下步骤操作：

  1. 实现算法类：在 algorithms.py 中创建新类，继承或实现统一的接口方法
  2. 注册算法：在 ALGORITHM_REGISTRY 中添加新条目
  3. 更新界面：如需自定义参数 UI，在 app.py 的算法实验室页面中添加对应组件

示例：添加随机森林算法

1) 创建 RandomForestModel 类，实现 train()、predict()、get_params()
2) 在注册表中添加："random_forest": {"name": "随机森林", "class": RandomForestModel, "type": "classification", ...}
3) 界面下拉菜单将自动包含新算法（如通过注册表动态生成选项）

第5章 界面设计

5.1 布局方案

界面采用单页多标签布局方案：

  • 顶部：平台标题栏，居中显示 "ML-Lab 机器学习可视化实验平台"
  • 中部：4 个功能页面通过标签切换显示，每次仅展示一个页面
  • 底部：每个页面内部采用上下布局，上方为输入控件区域，下方为结果展示区域

5.2 页面切换机制

页面切换采用纯 CSS 显隐控制实现，而非多页面路由：

  • 每个功能页面对应一个 Gradio gr.Column 容器
  • 通过 JavaScript/CSS 控制各容器的 display 属性实现显隐切换
  • 导航按钮触发页面切换事件

该方案的优势：

  • 无需页面刷新，切换流畅
  • 状态保持在内存中，切换页面不丢失数据
  • 实现简单，不依赖前端路由框架

ML-Lab（Machine Learning Laboratory）是为机器学习教学和实验而开发的可视化实验平台。项目旨在为职业院校和高校的机器学习课程提供一个零代码门槛、全流程可视化的实验环境，让学生无需编写代码即可体验数据预处理、算法训练、模型评估的完整流程。

2.3 数据流

平台的核心数据流如下：

  1. 用户在数据工作台选择数据集 → preprocessing.py 加载数据 → 展示预览和统计信息
  2. 用户配置预处理参数 → preprocessing.py 执行缺失值处理/特征缩放 → visualization.py 生成对比图
  3. 用户划分数据集 → preprocessing.py 返回训练集和测试集
  4. 用户在算法实验室选择算法和参数 → algorithms.py 通过工厂函数创建算法实例 → 调用 train() 训练模型
  5. 训练完成后 → evaluation.py 计算评估指标 → visualization.py 生成结果图表
  6. 用户在AI 助教提问 → llm_assistant.py 构建请求 → 调用通义千问 API → 返回回答

第4章 算法注册表设计

4.1 注册表结构

算法注册表 ALGORITHM_REGISTRY 是一个字典，键为算法名称，值为算法配置信息：

ALGORITHM_REGISTRY = {
    "linear_regression": {
        "name": "线性回归",
        "class": LinearRegressionModel,
        "type": "regression",
        "description": "使用最小二乘法拟合线性关系",
        "default_params": { ... }
    },
    "logistic_regression": {
        "name": "逻辑回归",
        "class": LogisticRegressionModel,
        "type": "classification",
        ...
    },
    ...
}

注册表字段说明

字段类型说明
namestr算法的中文显示名称
classtype算法实现类的引用
typestr算法类型："classification" 或 "regression"
descriptionstr算法简要描述
default_paramsdict默认超参数配置

4.2 工厂函数

create_algorithm() 工厂函数根据算法名称创建实例：

def create_algorithm(algorithm_name, **kwargs):
    if algorithm_name not in ALGORITHM_REGISTRY:
        raise ValueError(f"未知算法: {algorithm_name}")
    config = ALGORITHM_REGISTRY[algorithm_name]
    params = {**config["default_params"], **kwargs}
    return config["class"](**params)

工厂函数的设计优势：

  • 调用方无需知道具体类名，只需传入算法标识符
  • 自动合并默认参数与用户自定义参数
  • 集中管理算法的创建逻辑，新增算法只需更新注册表

4.3 扩展新算法

如需新增算法，按以下步骤操作：

  1. 实现算法类：在 algorithms.py 中创建新类，继承或实现统一的接口方法
  2. 注册算法：在 ALGORITHM_REGISTRY 中添加新条目
  3. 更新界面：如需自定义参数 UI，在 app.py 的算法实验室页面中添加对应组件

示例：添加随机森林算法

1) 创建 RandomForestModel 类，实现 train()、predict()、get_params()
2) 在注册表中添加："random_forest": {"name": "随机森林", "class": RandomForestModel, "type": "classification", ...}
3) 界面下拉菜单将自动包含新算法（如通过注册表动态生成选项）
