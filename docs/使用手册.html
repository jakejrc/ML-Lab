第1章 平台概述

1.1 平台简介

ML-Lab 是一个基于 Python 的机器学习可视化实验平台，集成了数据预处理、算法训练、模型评估和 AI 辅助教学四大核心功能。平台采用 Gradio 构建 Web 交互界面，后端基于 Scikit-learn 实现六大经典机器学习算法，并接入了通义千问大语言模型提供智能问答辅导。

本平台面向机器学习初学者和教学场景设计，具有以下特点：

  • 全流程可视化：从数据加载、预处理、模型训练到结果评估，每一步均配有交互式图表
  • 低门槛操作：无需编写代码，通过界面选择即可完成机器学习全流程实验
  • 六大经典算法：覆盖线性回归、逻辑回归、KNN、决策树、SVM、神经网络
  • AI 助教：集成通义千问大模型，支持多轮对话式学习辅导
  • Docker 部署：支持容器化一键部署，便于教学环境快速搭建

1.2 功能架构

平台采用模块化架构，由以下核心模块组成：

模块文件功能说明
主界面app.pyGradio Web 界面，页面路由与交互逻辑
算法引擎ml_lab/algorithms.py6 大算法封装、统一注册表与工厂函数
数据处理ml_lab/preprocessing.py数据集加载、缺失值处理、特征缩放、数据集划分
模型评估ml_lab/evaluation.py分类与回归评估指标计算与报告生成
可视化ml_lab/visualization.py10+ 种可视化图表生成函数
AI 助教ml_lab/llm_assistant.py通义千问 API 集成、多轮对话管理

1.3 界面布局

启动平台后，浏览器自动打开主界面。界面采用蓝白主题设计，包含以下区域：

  • 顶部标题栏：显示平台名称 "ML-Lab 机器学习可视化实验平台"
  • 页面导航栏：4 个功能页面标签 — 数据工作台、算法实验室、模型评估、AI助教
  • 内容区域：当前页面的交互表单、按钮和结果展示区

提示

页面导航采用纯 CSS 实现的标签切换机制，点击导航栏按钮即可在不同功能页面间切换，无需页面刷新。

2.3 缺失值处理

平台支持模拟缺失值并对其进行处理，用于教学演示缺失值处理策略：

操作步骤

  1. 在"缺失值比例"输入框中设置缺失值注入比例（0~1 之间）
  2. 选择缺失值处理策略：
    • 删除 (drop)：直接移除包含缺失值的行
    • 均值填充 (mean)：用该列均值填充缺失值
    • 中位数填充 (median)：用该列中位数填充缺失值
    • 众数填充 (mode)：用该列众数填充缺失值
  3. 点击"处理缺失值"按钮，查看处理前后的对比结果

说明

缺失值处理功能会在原始数据副本上操作，不会影响原始数据集。处理前后会展示对比可视化图表。

2.4 特征缩放

特征缩放可将不同量纲的特征归一化，提高模型训练效果：

缩放方法原理适用场景
标准化 (StandardScaler)将特征转换为均值 0、标准差 1 的分布大多数算法的默认选择
归一化 (MinMaxScaler)将特征缩放到 [0, 1] 区间对数据分布无假设时使用
最大绝对值缩放 (MaxAbsScaler)按最大绝对值缩放至 [-1, 1]保留稀疏性时使用
鲁棒缩放 (RobustScaler)基于中位数和四分位距缩放数据含有异常值时使用

2.5 数据集划分

在训练模型前，需将数据集划分为训练集和测试集：

  • 训练集比例：默认 80%，可调节（推荐 60%~90%）
  • 随机种子：默认 42，设置固定值可保证实验可复现
  • 划分完成后显示训练集和测试集的样本数量

提示

固定随机种子可以确保每次划分结果一致，便于对比不同算法在同一数据上的表现。完成数据集划分后，即可切换到"算法实验室"页面进行模型训练。

3.2 训练模型

操作步骤

  1. 确保已在"数据工作台"完成数据集加载和划分
  2. 切换到"算法实验室"页面
  3. 在下拉菜单中选择目标算法
  4. 根据需要调整算法超参数（见 3.3 节）
  5. 点击"开始训练"按钮
  6. 等待训练完成，查看训练结果和可视化图表

说明

平台内部通过 ALGORITHM_REGISTRY 注册表和 create_algorithm() 工厂函数统一管理所有算法实例。每个算法类均实现了标准接口：train()、predict()、get_params()，确保操作流程一致。

第4章 模型评估

模型评估页面提供全面的模型性能分析工具，包括评估指标计算和多维度可视化图表。

4.1 评估指标

分类任务指标

指标含义取值范围
准确率 (Accuracy)预测正确的样本占总样本的比例[0, 1]
精确率 (Precision)正类预测中真正为正的比例[0, 1]
召回率 (Recall)真正为正的样本中被正确预测的比例[0, 1]
F1 分数 (F1-Score)精确率与召回率的调和平均值[0, 1]

平台还生成完整的分类报告，包含每个类别的精确率、召回率、F1 分数和支持数。

回归任务指标

指标含义理想值
均方误差 (MSE)预测值与真实值之差的平方的均值越小越好
均方根误差 (RMSE)MSE 的平方根，与原始数据同量纲越小越好
平均绝对误差 (MAE)预测值与真实值之差的绝对值的均值越小越好
决定系数 (R²)模型解释的方差占总方差的比例越接近 1 越好

4.3 结果解读

如何判断模型性能

  • 分类任务：准确率 > 0.9 通常表示优秀的分类效果；关注 F1 分数可平衡精确率和召回率
  • 回归任务：R² > 0.8 表示模型拟合效果良好；RMSE 与数据量纲一致，便于直观理解误差大小

如何优化模型

  1. 调整超参数：返回算法实验室，修改算法参数后重新训练
  2. 更换算法：不同算法对不同数据类型的适应性不同，可尝试多种算法进行对比
  3. 数据预处理：返回数据工作台，尝试不同的特征缩放方法或缺失值处理策略
  4. 调整数据划分：修改训练集/测试集比例，确保数据量充足

第5章 AI 助教

5.1 功能简介

AI 助教模块集成了通义千问大语言模型，为用户提供机器学习知识的智能问答服务。该模块具备以下特点：

  • 多轮对话：支持连续提问，AI 助教能理解上下文语境
  • 内置知识库：预置 6 个机器学习核心概念的专业知识
  • 预设问题：提供 8 个常见问题快速入口

使用 AI 助教需要配置通义千问 API Key。如未配置，AI 助教功能将不可用。请参考《安装手册》完成 API Key 配置。

5.3 自由提问

操作步骤

  1. 切换到"AI助教"页面
  2. 在输入框中输入机器学习相关问题（支持中文）

AI 助教的系统提示词内置了以下 6 个核心知识领域的专业定义：

  3. 模型评估与选择（交叉验证、偏差-方差权衡、评估指标）

  5. 过拟合与正则化（L1/L2 正则化、Dropout、早停）

提示

AI 助教基于通义千问模型生成回答，适用于学习和理解机器学习概念。对于精确的数学推导或最新的研究进展，建议结合教材和论文进行深入学习。

第6章 常见问题与技巧

6.1 常见问题

Q1：平台启动后无法在浏览器中打开？

检查端口 7860 是否被占用。可通过以下方式指定其他端口启动：

python app.py --port 7861

Q2：训练模型时报错"数据未准备"？

请先在"数据工作台"中完成数据集加载和数据集划分操作，再切换到"算法实验室"进行训练。

Q3：AI 助教返回错误信息？

常见原因及解决方案：

  • API Key 未配置或已过期 → 参考安装手册重新配置
  • 网络连接问题 → 检查网络是否能访问通义千问 API 服务
  • 请求频率过高 → 等待片刻后重试

Q4：不同算法在同一数据集上表现差异很大？

这是正常现象。不同算法对数据分布的假设不同，适合的数据类型也不同。建议：

  • 对比多个算法的评估指标，选择最优方案
  • 通过数据可视化了解数据分布特征
  • 调整超参数进行调优

Q5：Docker 部署后无法访问？

确保容器端口映射正确，默认映射为宿主机 7860 → 容器 7860。检查防火墙设置是否允许该端口通信。

6.2 使用技巧

场景建议操作
